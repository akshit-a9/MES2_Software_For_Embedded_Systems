Data Cleaning

Strip units from strings

Sensor data often arrives as "8GB" or "1.5Kg". Strip the unit, cast to number.

python
df['Ram'] = (
    df['Ram']
    .str.replace('GB', '', case=False, regex=False)   # "8GB" → "8"
    .str.strip()                                          # remove whitespace
    .astype(int)                                          # "8" → 8
)

df['Weight'] = (
    df['Weight']
    .str.replace('kg', '', case=False, regex=False)
    .str.strip()
    .astype(float)
)

Handle missing values

python
from sklearn.impute import SimpleImputer

# Numerical: fill with median (robust to outliers)
imp_num = SimpleImputer(strategy='median')
df[num_cols] = imp_num.fit_transform(df[num_cols])

# Categorical: fill with most frequent
imp_cat = SimpleImputer(strategy='most_frequent')
df[cat_cols] = imp_cat.fit_transform(df[cat_cols])

# Or just fill manually
df['CPU_GHz'] = df['CPU_GHz'].fillna(df['CPU_GHz'].median())

Drop columns

python
df.drop(columns=['ID', 'raw_text_col'], inplace=True)

# Drop ALL remaining object (string) columns in one shot
object_cols = df.select_dtypes(include=['object']).columns
df = df.drop(columns=object_cols)
03

Scaling — Which One & When

Rule: Scale numerical features before any linear model or distance-based model. Tree-based models (Decision Tree, Random Forest, XGBoost) don't need it, but it never hurts.

StandardScaler — default choice

When: Numerical columns with different scales, data is roughly normally distributed, using linear models (Ridge, SVM, logistic regression).

Formula
z = (x − μ) / σ

After transform: mean = 0, std = 1. Values are still unbounded (can go < 0 or > 1).

python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()

# Fit on TRAINING data only, then transform both sets
X_train_scaled = scaler.fit_transform(X_train)  # learns μ, σ
X_test_scaled  = scaler.transform(X_test)       # uses same μ, σ

# Column-by-column (like in the notebook)
for col in num_cols:
    s = StandardScaler()
    df[col] = s.fit_transform(df[[col]])

MinMaxScaler — bounded output

When: You need output in [0, 1] (e.g., feeding into a neural network or TinyML model). No large outliers in data.

Formula
x' = (x − x_min) / (x_max − x_min)

After transform: min = 0, max = 1. A single outlier destroys this — use RobustScaler instead.

python
from sklearn.preprocessing import MinMaxScaler

scaler = MinMaxScaler()            # default range: [0, 1]
scaler = MinMaxScaler(feature_range=(-1, 1))  # custom range

X_scaled = scaler.fit_transform(X_train)

RobustScaler — outlier-resistant

When: Data has outliers (sensor spikes, bad readings). Uses median and IQR instead of mean/std.

Formula
x' = (x − median) / IQR     IQR = Q75 − Q25
python
from sklearn.preprocessing import RobustScaler

scaler = RobustScaler()
X_scaled = scaler.fit_transform(X_train)

Log Transform — skewed distributions

When: Data is heavily right-skewed (price, income, power consumption). Compresses the tail.

Formula
x' = log(x + 1)     (+1 avoids log(0))
python
df['Price_log'] = np.log1p(df['Price'])  # log(x + 1)

# Reverse when you need actual price back
predicted_price = np.expm1(model.predict(X_test))

Quick Decision Table

Situation Scaler Why
Normal-ish distribution, linear modelStandardScalerCenters data, equal variance
Need [0,1] output, clean dataMinMaxScalerBounded range
Outliers present (sensor spikes)RobustScalerUses median/IQR
Right-skewed (prices, power)np.log1p()Compresses tail
Tree models (RF, XGBoost)None neededTrees are scale-invariant
Critical: Always fit on training data only. Then transform both train and test. If you fit on test data, you've leaked future information into your model.
04

Encoding Categorical Variables

ML models need numbers. Categorical columns (object dtype) must be converted. Choice depends on cardinality (number of unique values) and the model type.

One-Hot Encoding (OHE)

When: Low cardinality (< ~20 unique values), no natural ordering, using linear/tree models.

How it works: Creates one binary column per category. "TypeName=Gaming" → column TypeName_Gaming = 1, all others = 0.

python
from sklearn.preprocessing import OneHotEncoder
import pandas as pd

enc = OneHotEncoder(sparse_output=False, handle_unknown='ignore')
encoded = enc.fit_transform(df[['TypeName']])
enc_df  = pd.DataFrame(encoded, columns=enc.get_feature_names_out(['TypeName']))
df = pd.concat([df, enc_df], axis=1)

# Shortcut with pandas (same result)
df = pd.get_dummies(df, columns=['TypeName'], drop_first=True)

Target Encoding

When: High cardinality (> 20 unique values), category has a relationship with target (e.g., Apple laptops cost more). Works well with linear models.

How it works: Replaces category with the mean target value for that category. "Apple" → mean price of Apple laptops.

python
import category_encoders as ce

enc = ce.TargetEncoder(cols=['Cpu'])
df['CPU_TE'] = enc.fit_transform(df[['Cpu']], df['Price'])

# Manual equivalent (mean encoding)
means = df.groupby('Company')['Price'].mean()
df['Company_TE'] = df['Company'].map(means)

Frequency Encoding

When: High cardinality, no strong target relationship, or you suspect rarer categories matter. Replaces category with its frequency proportion in the dataset.

python
def frequency_encode(df, col):
    freq = df[col].value_counts(normalize=True)  # proportions
    df[col + '_freq'] = df[col].map(freq)
    return df

df = frequency_encode(df, 'Company')

Quick Decision Table

Situation Method
Few categories (<20), linear/tree modelOne-Hot Encoding
Many categories (>20), target correlationTarget Encoding
Many categories, rarity mattersFrequency Encoding
Natural order (low/medium/high)Ordinal Encoding (0,1,2)
Tree model, don't care muchTarget or Frequency — both fine

Feature Engineering

Raw data is rarely in the best form for a model. Feature engineering = creating new, more informative columns.

Parse structured info out of strings

A column like "Intel Core i5 2.3GHz" contains four signals: brand, family, model, clock speed. Extract them separately.

python
import re

def parse_cpu(cpu_str):
    if pd.isna(cpu_str):
        return pd.Series([None]*4, index=['Brand','Family','Model','GHz'])
    tokens = cpu_str.split()

    brand   = tokens[0]                                        # "Intel"
    family  = tokens[1] if len(tokens) > 1 else None            # "Core"
    ghz_m  = re.search(r'(\d+\.?\d*)GHz', cpu_str)
    ghz    = float(ghz_m.group(1)) if ghz_m else None
    model  = None
    for t in reversed(tokens):
        if re.search(r'\d', t) and 'GHz' not in t:
            model = t; break

    return pd.Series([brand, family, model, ghz],
                     index=['Brand','Family','Model','GHz'])

df[['CPU_Brand','CPU_Family','CPU_Model','CPU_GHz']] = \
    df['Cpu'].apply(parse_cpu)

Ensures each price bracket has proportional representation in both splits. Essential when your dataset is small or skewed.

from sklearn.model_selection import StratifiedShuffleSplit

split = StratifiedShuffleSplit(n_splits=1, test_size=0.2, random_state=42)

for train_idx, test_idx in split.split(df, df['price_cat']):
    train_set = df.iloc[train_idx]
    test_set  = df.iloc[test_idx]

# Separate features and labels

Models

Ridge — Regularized Linear Regression

Linear regression + L2 penalty to prevent overfitting. Good baseline. Needs scaling.

from sklearn.linear_model import Ridge
from sklearn.model_selection import GridSearchCV

ridge = Ridge()

Random Forest — Ensemble of Decision Trees

Builds many trees on random subsets, averages their outputs. Robust, handles non-linearity, feature importance built-in. Scale-invariant.

python
from sklearn.ensemble import RandomForestRegressor
from sklearn.pipeline import make_pipeline
from sklearn.model_selection import cross_val_score

rf = RandomForestRegressor(random_state=42)
rf_params = {
    'n_estimators': [100, 200],       # number of trees
    'max_depth': [5, 10, None],        # None = grow until pure
    'min_samples_split': [2, 5]
}
rf_grid = GridSearchCV(rf, rf_params, cv=10,
                       scoring='neg_mean_squared_error', n_jobs=-1)
rf_grid.fit(X_train, y_train)
preds = rf_grid.predict(X_test)

XGBoost — Gradient Boosted Trees

Builds trees sequentially, each correcting the last. State of the art for tabular data. Scale-invariant.

python
from xgboost import XGBRegressor
from sklearn.preprocessing import StandardScaler

# Optional: scale the target (helps convergence)
tgt_scaler = StandardScaler()
y_scaled = tgt_scaler.fit_transform(y_train.values.reshape(-1, 1)).ravel()

model = XGBRegressor()
model.fit(X_train, y_scaled)

raw_preds = model.predict(X_test)
preds = tgt_scaler.inverse_transform(raw_preds.reshape(-1, 1))

Pipeline — keep it clean

Chain preprocessing + model. Prevents data leakage and simplifies deployment.

python
from sklearn.pipeline import make_pipeline

pipe = make_pipeline(
    StandardScaler(),
    RandomForestRegressor(n_estimators=100)
)
pipe.fit(X_train, y_train)
preds = pipe.predict(X_test)

# Cross-validate the whole pipeline safely
scores = cross_val_score(pipe, X_train, y_train,
                          cv=10, scoring='neg_mean_squared_error')
rmse = np.sqrt(-scores)
print(f"Mean RMSE: {rmse.mean():.0f} ± {rmse.std():.0f}")

Evaluation

RMSE — Root Mean Squared Error

Same unit as target. Penalizes large errors heavily. Lower = better.

Formula
RMSE = √( (1/n) · Σ (y_pred − y_true)² )
python
from sklearn.metrics import mean_squared_error

rmse = np.sqrt(mean_squared_error(y_test, preds))
print(f"RMSE: {rmse:.2f}")

Average Percentage Difference

More intuitive than RMSE — tells you the average % error relative to the actual value. Domain-agnostic.

Formula
APD = mean( |y_pred − y_true| / ((y_pred + y_true) / 2) ) × 100
python
def avg_pct_diff(actual, predicted):
    diff    = np.abs(actual - predicted)
    avg     = (actual + predicted) / 2
    return  np.mean(diff / avg * 100)

print(f"Ridge APD:  {avg_pct_diff(y_test, preds_ridge):.2f}%")
print(f"RF APD:     {avg_pct_diff(y_test, preds_rf):.2f}%")
print(f"XGBoost APD:{avg_pct_diff(y_test, preds_xgb):.2f}%")

Cross-validation — don't trust a single split

python
scores = cross_val_score(
    model, X_train, y_train,
    cv=10,                              # 10-fold
    scoring='neg_mean_squared_error'    # sklearn returns negative MSE
)
rmse_cv = np.sqrt(-scores)
print(f"CV RMSE: {rmse_cv.mean():.0f} ± {rmse_cv.std():.0f}")
# High std = model is unstable / overfitting on some folds
For embedded/TinyML: RMSE tells you raw error magnitude. APD tells you relative error — e.g., "my model is 12% off on average." For a battery SoC estimator, 12% off is catastrophic; for a price recommender, it might be acceptable. Always sanity-check with domain context.
